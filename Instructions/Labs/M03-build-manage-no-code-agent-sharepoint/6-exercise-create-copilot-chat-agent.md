Dans cet exercice, vous créez votre propre agent personnalisé pour Microsoft 365 Copilot Chat. Les agents personnalisés sont des assistants IA personnalisés conçus pour faciliter des flux de travail, des tâches ou des domaines spécifiques au sein de votre organisation. Ils peuvent également vous aider avec un projet privé qui n’est pas lié au travail. En concevant un agent, vous devez explorer comment guider le comportement et les réponses de Copilot à l’aide d’options de configuration telles que les rôles, les instructions, les sources de données et les autorisations. Cet exercice a pour but de vous aider à comprendre comment réfléchir de manière critique sur la finalité et les fonctionnalités des agents que vous créez dans Microsoft 365 Copilot Chat.

### Exercice

Dans cet exercice, vous devez faire preuve de créativité et concevoir un agent qui présente un intérêt particulier pour vous. Par exemple, vous pouvez résoudre un problème d’entreprise réel, améliorer la productivité dans un domaine spécifique de votre entreprise ou traiter un sujet personnel qui vous intéresse. Étant donné que chaque étudiant crée son propre agent personnel, les instructions ci-dessous portent essentiellement sur la structure et l’ordre de configuration d’un agent plutôt que de spécifier des valeurs réelles à saisir dans chaque champ. C’est à vous de décider de ce que vous voulez saisir dans chaque champ lorsque vous créez votre agent.

1. Dans **Microsoft Edge**, ouvrez un nouvel onglet et entrez l’URL suivante : [**https://M365copilot.com**](https://M365copilot.com)
1. Dans **Microsoft 365**, sélectionnez **Créer un agent** dans le volet de navigation.
1. Dans la fenêtre **Copilot Studio**, l’onglet **Décrire** s’affiche par défaut. Si vous souhaitez créer votre agent à partir d’un modèle existant, vous pouvez sélectionner l’un des modèles affichés ou sélectionner **Afficher tous les modèles**. Si vous avez sélectionné **Afficher tous les modèles**, faites défiler la liste des modèles et sélectionnez celui de votre choix. Si vous avez sélectionné un modèle, notez les invites de démarrage qui sont affectées à votre agent à partir du modèle sélectionné.
1. Le champ **Taper votre message** qui apparaît sous les modèles, sur le côté gauche de la page, est utilisé pour configurer votre agent. Le champ **Taper votre message** qui apparaît sur le côté droit de la page, sous les invites de démarrage, est utilisé pour tester votre agent. Étant donné que vous configurez l’agent, les étapes restantes utilisent le champ **Taper votre message** qui apparaît sur le côté gauche de la page. Dans ce champ, écrivez un bref résumé qui décrit la fonction de votre agent, puis sélectionnez l’icône **Envoyer**(flèche orientée vers l’avant).
1. Si vous n’avez pas indiqué de nom pour votre agent dans la description que vous venez de saisir, Copilot Studio propose généralement un nom suggéré et vous demande soit de le confirmer, soit d’en indiquer un nouveau. Vous devez répondre de manière appropriée. 
1. À ce stade, Copilot Studio vous demande généralement d’affiner les instructions de votre agent. Il peut le faire par le biais d’une série de questions. Répondez à chaque question de manière appropriée jusqu'à obtenir une description de l’agent qui soit à votre convenance. Copilot Studio peut également vous demander si vous souhaitez ajouter des sites Web disponibles publiquement en tant que sources de données pour cet agent. Vous pouvez certes répondre ici, mais vous attribuez généralement vos sources de connaissances sous l’onglet **Configurer** dans les étapes à venir. 
1. Une fois que vous avez finalisé la description de l’agent, sélectionnez l’onglet **Configurer** en haut du formulaire. 
1. Sous l’onglet **Configurer**, vérifiez les champs **Nom**, **Description** et **Instructions**. Apportez les modifications qui vous semblent nécessaires. Notez que les instructions ont été créées par Copilot Studio pour traiter la **Description** que vous avez fournie, ainsi que les réponses aux questions de suivi auxquelles vous avez répondu sous l’onglet **Décrire**. Ne perdez pas de vue que les instructions ont tendance à être beaucoup plus précises par nature que la description que vous avez saisies. Vérifiez que les instructions reflètent votre intention quant à la manière dont votre agent doit se comporter et répondre aux utilisateurs. Apportez les modifications, si nécessaire.
1. Dans la section **Connaissances**, vous devez joindre toutes les sources de données pertinentes, telles que les sites SharePoint, les documents ou les intégrations tierces, que votre agent doit référencer.

    > [!WARNING]
    > Si vous n’ajoutez pas de sites Web publics ou de données professionnelles à votre agent, il peut toujours répondre à vos demandes. Toutefois, il utilisera uniquement les fonctionnalités générales du modèle LLM sous-jacent, telles que la réponse aux questions courantes sur les connaissances, le raisonnement de base ou la génération de texte généralisé. Il est donc recommandé d’ancrer vos agents dans des données de travail spécifiques ou des sites Web publics. Dans le cas contraire, les réponses de l’agent pourraient ne pas s’inscrire dans le contexte ni être pertinentes pour des tâches spécifiques à l’entreprise.

1. Dans la section **Fonctionnalités**, vous pouvez éventuellement activer l’**Interpréteur de code** et les outils de **Générateur d’images** en sélectionnant leurs commutateurs bascules. Les deux outils sont désactivés par défaut.
1. Dans la section **Invites de démarrage**, Copilot Studio affiche les six invites de démarrage qu’il a créées automatiquement lorsque vous avez entré la description de l’agent sous l’onglet **Décrire**. Vous pouvez modifier ou supprimer l’une des invites existantes ou en ajouter de nouvelles.
1. Sur le côté droit de la page, sous les invites de démarrage, vous pouvez éventuellement tester l’agent en saisissant un message dans le champ d’invite. Vous pouvez tester votre agent aussi longtemps que vous le souhaitez et affiner votre configuration si elle ne fonctionne pas ou ne répond pas comme il le devrait. 
1. Lorsque vous estimez que l’agent est configuré correctement, sélectionnez le bouton **Créer** en haut de la page.  
1. Une fois l’agent créé, la fenêtre **Votre agent a été créée avec succès** s’affiche. Si vous souhaitez modifier les paramètres de partage, sélectionnez le lien **Modifier les paramètres de partage**. Cette opération vous permet de définir qui dans votre organisation peut utiliser cet agent en configurant les paramètres d’accès utilisateur. Par défaut, l’agent fonctionne uniquement pour vous. Une fois que vous avez configuré les paramètres de partage (ou que vous les avez laissés comme vous le souhaitez), sélectionnez le bouton **Accéder à l’agent** pour ouvrir votre agent. 
1. Dans la fenêtre de votre agent, notez comment l’agent apparaît dans le volet de navigation. N’hésitez pas à saisir l’une des invites de démarrage ou à saisir d’autres invites personnalisées pour tester l’agent. 
1. Lorsque vous avez terminé d’utiliser l’agent, sélectionnez **Tous les agents** dans le volet de navigation. Dans la fenêtre **Magasin d’assistants** qui apparaît, notez comment votre agent s’affiche dans la section **Vos agents**. Vous pouvez accéder à cet agent à partir du Magasin d’assistants ou sélectionner l’icône en forme d’épingle qui s’affiche à côté de l’agent dans le volet de navigation si vous souhaitez que l’agent s’affiche toujours sous la liste des agents dans le volet.
